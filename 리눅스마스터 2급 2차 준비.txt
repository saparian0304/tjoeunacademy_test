bash 셸 관련 설정 파일들
/etc/profile 
 - 모든 사용자의 셀 환경을 ㅈ제어하는 전역적인 시스템 설정파일
 - 환경변수와 bash가 수행시 실행되는 프로그램을 제어
 - 관리자만 설정 가능해야 하며 모든 사용자들에게 반영되어야함
/etc/bashrc
 - 별칭(alias)와 bash가 수행시 실행되는 함수를 전역적으로 제어
 - 생략되기도 하며 /etc//profile에 내용이 포함되기도 함
~/.bash_profile
 - 개인 사용자의 셸 환경을 제어하는 지역적인 시스템 설정파일
 - PATH, 환경 변수 등을 설정 또는 변경할 때 사용
 - 로그인시 로딩
~/.bash_history
 - 명령어의 재사용이 가능하게 함
 - 위/아래 화살표를 사용하여 하용했던 명령어 검색 가능
~/.bashrc
 - 별칭(alias)와 bash가 수행시 실행되는 함수를 지역적으로 제어
~/.bash_logout
 - 로그아웃하기 직전에 실행되는 시스템 설정파일

LVS(Linux Virtual Server, 부하분산 클러스터)
 - 실제 서버를 통해 IP 로드 밸런스를 맞추기 위한 통합된 소프트웨어 구성요소 모음

HA(High Availability, 고가용성 클러스터)
 - 긴 시간동안 지속적으로 운영이 가능한 시스템이나 컴포넌트를 가리킨다. 고가용성 클러스터는 로드밸런서와 백업시스템을 이용하여 지속적인 서비스를 제공한다

HPC(High Performance Computing, 고계산성 클러스터)
 - 고성능의 계산능력을 제공하기 위한 목적으로 여러대의 컴퓨터를 병렬로 묶어서 슈퍼 컴퓨터를 제작하는 방식


LVM 관련 용어
 - PV(Physical Volume) : 디스크를 LVM에서 사용할 수 있게 변환하는 작업
 - VG(Volume Group) : PV가 모여 만들어진 그룹, VG는 다시 LV로 할당할 수 있는 공간이기도 함
 - LV(Logical Volume) : VG에서 사용자가 필요한 만큼 할당되서 만들어지는 공간
 - PE(Physical Extent) : PV에서 나누어 사용하는 블록. 4MB 단위
 - LE(Logical Extent) : LV가 나누어진 일정한 크기의 블록. PE와 서로 1:1 대응

vi 편집기 단어 치환 명령
s/<SEARCH>/<REPLACE>/g
 - SEARCH에 해당되는 문자열을 찾아서 REPLACE로 치환
 - g : 한 행에 같은 내용이 여러번 나타나도 모두 치환
 - <> : 완벽하게 같은 경우에만 바꾸라는 것(문자단위)
 - 특수문자는 앞에 \표시로 구분해줘야한다.

x 윈도 개념과 특징
 - 리눅스 환경의 각종 애플리케이션과 유틸리티에 대해 그래픽 사용자 인터페이스 제공

 - 1987 오픈 소스프로젝트 하에 X11 발표

X 윈도 특징
 - 네트워크 기반 그래픽 환경 지원
 - 이기종 시스템 사이에서도 사용 가능
 - 그래픽 환경 자원들이 특정 형태로 정의되어 있지 않음
 - 디스플레이 장치에 의존적이지 않으며 원하는 인터페이스를 만들 수 있음

X 윈도 시스템
 IBM과 MIT, DEC 공동의 아데나 프로젝트를 통해서 Bob Scheifler와  Jim Gettys가 1984년에 처음 개발
 1988년에는 수많은 컴퓨터 제조업체로 이루어진 X컨소시엄이 조직되고, X컨소시엄에 의해서 X11 버전이 처음으로 개정되어 X11R2가 발표되었으며 1996년에는 최종 개정판인 X11R6 버전을 내놓으면서 X11R6.3
